1. PROJECT OVERVIEW
================================================================================

Vibe3D is an AI-powered 3D scene accelerator that bridges natural language
commands to Unity Editor actions via MCP (Model Context Protocol). It provides
a web-based interface for controlling Unity scenes, analyzing 3D assets,
running drone photogrammetry pipelines, and extracting building information
from OBJ tile meshes.

- Web UI:    http://127.0.0.1:8091
- API Docs:  http://127.0.0.1:8091/docs
- WebSocket: ws://127.0.0.1:8091/ws
- MCP:       http://localhost:8080/mcp (Unity Editor)

Key Pipeline:
  User (Korean/English) --> NLU Engine --> Plan Generator --> Validator
  --> Approval UI --> Executor --> MCP Batch --> Unity Editor --> Undo Store

5.3 GeoBIM BUILDING EXTRACTION
--------------------------------------------------------------------------------

5-stage batch pipeline:
  Stage 0: Ingest OBJ tiles
  Stage 1: Collider proxy generation (Blender decimation)
  Stage 2: Building segmentation (RANSAC ground + DBSCAN clustering)
  Stage 3: Attribute extraction (OBB, footprint, roof planes)
  Stage 4: Export (JSONL + folder structure)

[geobim_models.py] 258 lines — Data Models
  - BuildingCandidate: id, tile_id, centroid, height, area, volume,
    footprint, confidence, tags, roof_planes, obb
  - RoofPlane: normal, d, area, tilt_deg, azimuth_deg
  - OBBData: center, axes, half_extents, rotation
  - SensorParams: position, height, yaw, pitch, hFOV, vFOV, maxDistance
  - ColliderProxyResult: input_path, output_path, original/simplified face count
  - PipelineState: stage tracking with timestamps
  - GeoBIMReport: extraction results with statistics

[geobim_extractor.py] 563 lines — Building Extraction Engine
  Algorithms:
    - RANSAC ground plane detection (configurable iterations, threshold)
    - DBSCAN / grid-based clustering in XZ plane
    - OBB computation via PCA on convex hull
    - Convex hull footprint extraction
    - Roof plane extraction (iterative RANSAC on upper 30% triangles)
    - Automatic roof_type tag classification:
        tilt < 5    -> flat
        tilt < 15   -> low_slope
        tilt < 35   -> gable
        else        -> shed

[geobim_db.py] 412 lines — SQLite Database
  Tables:
    buildings       — building_id, tile_id, centroid, heights, area, volume,
                      footprint, confidence, tags, obb, roof_planes
    collider_proxies — tile_id, input_path, output_path, face counts
    reports         — timestamp, status, JSON report data
    review_queue    — building_id, confidence, status, reviewer_label,
                      reviewed_at, notes (HITL workflow)

  Key Methods:
    save_buildings(), get_building(), get_buildings(), get_footprints()
    get_summary(), spatial_query(), clear_all()
    save_collider_proxy(), get_collider_proxies()
    save_report(), get_latest_report()
    populate_review_queue(), get_review_queue()
    review_building(), get_review_stats()

[geobim_pipeline.py] 199 lines — Batch Pipeline Orchestrator
  - Sequential stage execution with progress callbacks
  - Error handling per stage with status tracking

[geobim_collider_proxy.py] 252 lines — Collider Proxy Generator
  - Blender headless mesh decimation
  - Batch generation for all tiles
  - Blender availability check

[geobim_export.py] 129 lines — Data Export

[geobim_simulation.py] 639 lines — Spatial Analysis

  GridPathfinder (A* pathfinding):
    - Occupancy grid from building footprints
